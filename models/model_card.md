# TransitLens ML Core — Model Card

## 1. Model Type

**Primary:** Rule-based decision tree classifier operating on 11 physically-interpretable features extracted from Box Least Squares (BLS) transit detection results.

**Secondary (optional):** Scikit-learn Random Forest or XGBoost ensemble classifier. Activated only when trained model files (`rf_model.pkl` or `xgb_model.pkl`) are present and `ml_classifier.enabled: true` in `rule_config.yaml`.

When both classifiers are active:
- If predictions **agree**: the ML model's class is used with ML confidence.
- If predictions **disagree**: the rule-based prediction is used (prioritising interpretability), with disagreement noted in the explanation string.

## 2. Training Data

### Rule-based classifier
The rule-based classifier requires **no training data**. Its thresholds are derived from astrophysical constraints and observational statistics from published TESS planet catalogues.

### Synthetic evaluation data
Three synthetic TESS-like light curves generated by `transitlens-data-pipeline`:

| Case | True Class | Period | Depth | SNR | Transit Count |
|------|-----------|--------|-------|-----|---------------|
| Candidate A | exoplanet_like | 3.42 days | 1.3% | ~21σ | ~7 |
| Candidate B | eclipsing_binary_like | 1.87 days | 18% | ~180σ | ~14 |
| Candidate C | noise_or_other | — | — | <3σ | 0 |

**Key limitation:** Tuned and validated on synthetic data only. Real TESS light curves have more complex noise properties (instrumental systematics, stellar variability, blended sources).

## 3. Features Used

All 11 features are computed from the BLS detection result and the phase-folded light curve:

| # | Feature | Range | What it measures |
|---|---------|-------|------------------|
| 1 | `bls_power` | [0, 1] | Normalised BLS peak power — signal consistency |
| 2 | `snr` | [0, ∞) | Signal-to-noise ratio of transit detection |
| 3 | `period_days` | (0, ∞) | Best-fit orbital period from BLS |
| 4 | `duration_days` | (0, ∞) | Best-fit transit duration from BLS |
| 5 | `depth` | [0, 1) | Fractional flux drop at transit centre |
| 6 | `transit_count` | [0, ∞) | Number of expected transits in time series |
| 7 | `odd_even_depth_delta` | [0, ∞) | |depth_odd - depth_even| — EB discriminator |
| 8 | `v_shape_score` | [0, 1] | Transit shape: 0 = flat-bottomed, 1 = V-shaped |
| 9 | `local_noise` | (0, ∞) | RMS of out-of-transit phase-folded flux |
| 10 | `depth_to_noise_ratio` | [0, ∞) | depth / local_noise — local detection significance |
| 11 | `phase_shape_kurtosis` | (-∞, ∞) | Excess kurtosis of in-transit flux profile |

## 4. Rule Thresholds

All thresholds are defined in `models/rule_config.yaml` — none are hardcoded in Python.

### Detection gate
| Parameter | Value | Justification |
|-----------|-------|---------------|
| `bls_power_threshold` | 0.15 | Below this, BLS power is indistinguishable from noise |
| `snr_threshold` | 5.0 | Standard astronomical detection threshold |

### Classification decision tree
| Parameter | Value | Justification |
|-----------|-------|---------------|
| `depth_threshold_eb` | 0.050 (5%) | No planet can block >3% of a Sun-like star |
| `odd_even_threshold` | 0.020 | Significant depth alternation → binary eclipses |
| `v_shape_threshold` | 0.40 | V-shaped profiles indicate grazing stellar eclipses |
| `depth_snr_threshold` | 6.0 | Signal must exceed 6× local noise to be confident |

### Confidence scoring
Per-class weighted components with partial scoring. Full specification in `models/rule_config.yaml` under the `confidence` section.

## 6. Known Limitations

1. **Synthetic data only**: All thresholds tuned on 3 synthetic cases. Real TESS light curves have complex instrumental systematics (scattered light, momentum dumps, thermal drifts) not present in synthetic data.

2. **Conservative thresholds**: The depth threshold of 5% for EB classification may miss low-contrast eclipsing binaries. The SNR threshold of 5.0 may miss shallow transits in noisy data.

3. **Three classes only**: The classifier does not distinguish between:
   - Blended eclipsing binary (background EB diluted by the target star)
   - Stellar variability (rotational modulation, pulsation)
   - Systematic artefacts (instrumental trends)
   All of these are classified as `noise_or_other`.

4. **Single-sector limitation**: The pipeline analyses one TESS sector (27 days) at a time. Cannot detect planets with periods >13.5 days (need ≥2 transits). Multi-sector support is a stretch goal.

5. **No centroid analysis**: Cannot distinguish on-target transits from background eclipsing binaries without pixel-level centroid data.

6. **Phase aliasing**: The alias check is simple (×2 and ÷2 harmonics). More complex aliasing patterns (e.g., ×3, beat frequencies) are not checked.

## 7. How to Improve

1. **More labeled real TESS data**: Add confirmed planet and EB cases from the TESS Objects of Interest (TOI) catalogue. Retune thresholds on real data.

2. **Train RF/XGBoost**: Use expanded labeled dataset to train the optional ML classifier. Enable with `ml_classifier.enabled: true` in `rule_config.yaml`.

3. **CNN classifier**: Train a 1D CNN on phase-folded light curves directly — bypasses hand-crafted features.

4. **Multi-sector stitching**: Combine multiple TESS sectors to extend the time baseline and detect longer-period planets.

5. **Centroid analysis**: Add pixel-level centroid shift features for background EB discrimination.

6. **Confidence calibration**: Apply Platt scaling or isotonic regression to calibrate confidence scores against actual accuracy on a held-out test set.
